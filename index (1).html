Conditional Statements

Explanation

Conditional statements allow a program to make decisions based on a condition being true or false.
Using if / else, the program takes different paths depending on the input or situation.
Without conditionals, every program would always do the same thing — no logic, no decisions.
Real-life example: An exam system checks: if marks ≥ 40 → print "Pass", else print "Fail".

Loops

Explanation

A loop repeats a block of code multiple times without you having to write it again and again.
A for loop runs a fixed number of times; a while loop keeps running as long as a condition is true.
Loops are one of the most powerful tools in programming — they handle repetitive tasks automatically.
Real-life example: Sending a welcome email to 1000 users is done with a single loop — not 1000 lines of code.

Classes and Objects

Explanation

A class is a blueprint or template that defines properties and actions for a type of object.
An object is a real instance created from that blueprint — it has its own data and can perform actions.
This is the foundation of Object-Oriented Programming (OOP), used in almost every modern application.
Real-life example: "Car" is a class with properties like color and speed. BMW and Audi are objects — real instances of the Car class.

Built-in Modules / Libraries

Explanation

A module (or library) is a collection of pre-written code that you can import and use in your program.
Instead of writing complex logic from scratch, you simply import the module and call its functions.
Every language has a rich set of built-in modules for math, file handling, networking, dates, and more.
Real-life example: Instead of writing your own square root formula, you import the Math module and call math.sqrt() — done in one line.

Inheritance

Explanation

Inheritance allows a new class (child) to acquire the properties and methods of an existing class (parent).
This promotes code reuse — you don't rewrite what already exists; you just extend it.
The child class can also add its own new features or override the parent's behaviour.
Real-life example: A "Dog" class inherits from "Animal" — it automatically gets properties like name and age, and adds its own method like fetch().

File Handling

Explanation

File handling allows a program to save data permanently to a file on disk and read it back later.
Without file handling, all data is lost the moment the program closes — like writing on a whiteboard that gets erased.
You can open a file in different modes: read (r), write (w), or append (a).
Real-life example: A note-taking app saves your notes to a text file. When you reopen the app, it reads the file and restores everything you wrote.

Deadlock Detection and Resource Allocation System

Explanation

A deadlock happens when two or more processes are stuck waiting for each other forever.
Imagine Process A holds Resource 1 and waits for Resource 2, while Process B holds Resource 2 and waits for Resource 1 — neither can proceed.
Real-world example: Two trains on the same track facing each other, and neither can move.
This experiment simulates resource allocation and detects if a deadlock condition exists.
It uses the Banker's Algorithm concept to safely allocate resources.

Distributed Computing System using Remote Procedure Calls

Explanation

RPC (Remote Procedure Call) lets a program call a function that runs on a different machine over a network.
The calling program does not need to know where the function is — it just calls it like a local function.
Real-world example: A mobile banking app calls a function on the bank's remote server to check your balance.
This experiment simulates an RPC-style system where a client calls a remote add/multiply function.
It demonstrates distributed computing concepts fundamental to cloud services and APIs.
